Image Slideshow

Laboratory Part

  • Sample Application
  • Main window:
    • initial size: 800 x 600 px, minimum size: 800 x 600 px
    • appears in the center of the screen
    • main window contains:
      • menu at the top
      • left side of application contains two expanders:
        • Explorer - a tree view to explore and select (open) folder from the file system
        • File info - displays basic information about selected image file from the right side of application
      • right side of application - displays list of image files from the selected (opened) folder
  • Menu:
    • File
      • Open folder - opens selected folder and display image files from this folder in the right side of application
      • Exit - closes the application
    • About - displays short information about application
  • Explorer:
    • this is an expander with a tree view
    • this expander should be expanded by default
    • the tree view initially displays computer partitions
    • allows to expand each partition and display child folders, and then expand each folder etc.
    • user can select folder and its list of image files appears on the right side of app
  • File info:
    • this is an expander with a basic information about selected image file
    • this expander should not be expanded by default
    • displays basic information about selected image file or information: "No file selected"
    • the basic information is: name, width, height and size in KB
  • List of image files - right side of app:
    • displays image files from selected (opened) folder
    • each image file is represented by a simple tile with the file's name
    • each tile has 1px border
    • a single click on the tile displays information (in file info) about the image file represented by this tile
    • list of image files displays vertical scroll bar when necessary
  • Tips:
    • WindowStartupLocation, Window.MinHeight, Window.Height, Window.MinWidth, Window.Width
    • Menu, MenuItem
    • Expander, TreeView, FileInfo, Converter
    • Environment.GetLogicalDrives(), Directory.GetDirectories(), DirectoryInfo
    • ListView, ListView.ItemTemplate, DataTemplate
    • ListView.ItemsPanel, WrapPanel
    • Border
    • FolderBrowserDialog (System.Windows.Forms)
  • Note: In all doubtful and untold aspects application should behave like example app (except possible bugs)
  • Scoring:
    • menu + overall application layout: 2p
    • tree view explorer: 3p
    • open folder and display its content in the right side of application (as tiles): 3p
    • display "No file selected" and hide table with file info when no file is selected: 1p
    • display file info: 3p

Home Part

  • Sample Application
  • Meeting all the requirements of laboratory part is a precondition for further evaluation of the task
  • Main window:
    • main window is still splitted into 2 parts - left and right
    • there is a new expander in the left side of app - 'Slideshow'
    • the style of tiles in the right side of app is changed now
  • Menu:
    • new menu item 'Start slideshow' which contains available slideshow effects (loaded dynamically)
    • click on the slideshow effect starts the slideshow with selected effect
  • List of image files:
    • now each image file is represented by thumbnail consisting of icon (image preview) and file's name below the icon
    • each thumbnail has bottom and right side shadow effect
    • list of image files displays vertical scroll bar when necessary
  • Slideshow expander:
    • contains combobox with all avaible slideshow effects
    • contains button 'Start slideshow'
  • Slideshow:
    • when user click 'Start slideshow' in 'Slideshow expander' or select appropriate effect from the menu the slideshow starts
    • appears new window with fixed size (1024 x 768 px), without any border, and parent window is locked until the slideshow window is not closed
    • the new window is not displayed in the taskbar
    • the window with slideshow animation contains context menu which appears on right mouse button click:
      • it contains two options: 'Play/Pause slideshow' and 'Stop slideshow'
      • 'Play/Pause slideshow' pauses or resumes slideshow animation
      • 'Stop slideshow' closes slideshow window and unlock the main window
    • the slideshow consists of image files from selected (opened) folder
    • you should implement three transition effects:
      • horizontal effect - the images appear in the right edge and they are animated to the left edge
      • vertical effect - the images appear in the bottom edge and they are animated to the top edge
      • opacity effect - the opacity of previus image is animated to 0 and opacite of next image is animated to 1
      • please see example app to see how described effects should look like
  • System of slideshow effects (plugins):
    • you have to implement three slideshow effects (plugins): horizontal, vertical and opacity - described above
    • the plugin system should be able to add new functionality without recompiling the application, by inserting a new DLL into directory with exe file
    • the plugin can be implemented as a class library (compiled to DLL file) which implements the appropriate interface - sample interface is presented below: 
          public interface ISlideshowEffect
    {
        string Name { get; }
        void PlaySlideshow(Image imageIn, Image imageOut, double windowWidth, double windowHeight);
    }
    • the plugin interface should be implemented in a way that allows to share it with other developers
    • example plugin should be implemented in a way presented below: 
          public class SampleSlideshowEffect : ISlideshowEffect
    {
        public string Name
        {
            get
            {
                return "Name of Example Slideshow Effect";
            }
        }

        public void PlaySlideshow(Image imageIn, Image imageOut, double windowWidth, double windowHeight)
        {
            // Animate between two images in appropriate way
        }
    }
  • Tips:
    • DataTemplate, Style
    • DoubleAnimation, Opacity
    • Build Path in Project properties
  • Note: In all doubtful and untold aspects application should behave like example app (except possible bugs)

    • slideshow context menu: 2p
    • slideshow effects: 3p (1p for each effect)
